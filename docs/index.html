🎓 Student Management System

An interactive, full-featured system built with Python, SQLite, and Gradio to manage students, courses, enrollments, and performance reports.

📌 Project Overview

This Student Management System provides an intuitive UI to perform core academic operations including:

  • Managing student records (add, search, update GPA, delete)
  • Adding and listing academic courses
  • Enrolling students in multiple courses
  • Viewing enrolled courses per student
  • Generating insightful reports like average GPA by age, total students, etc.

🛠️ Key Features

🧑 Student CRUD (Create, Read, Update, Delete)
📘 Course Management
🔗 Many-to-Many Enrollments
✏️ GPA Updates
📊 SQL-based Reports (AVG, COUNT, GROUP BY)
🖥️ Gradio Frontend – No HTML/CSS needed

🧠 SQL Concepts Practiced

Perfect for beginners and intermediate learners to explore SQL in a real-world use case.

  • CREATE TABLE, INSERT, DELETE, UPDATE
  • SELECT, JOIN, WHERE, LIKE
  • GROUP BY, AVG(), COUNT()
  • PRIMARY KEY, FOREIGN KEY, constraints

🌐 Live UI Preview

This app runs locally or can be deployed to platforms like Railway, Render, or even GitHub Codespaces.

🚀 How to Run

  1. Clone the project: git clone https://github.com/yourusername/student-mgmt.git
  2. Install dependencies: uv pip install -r requirements.txt or pip install gradio
  3. Run it: python main.py
  4. Open browser: http://localhost:7860

📦 Deployment Notes

This system uses SQLite for simplicity and learning. For cloud deployment with persistent storage, we recommend switching to PostgreSQL and using platforms like:

  • 🔁 Railway (1-click PostgreSQL + Gradio support)
  • 🔧 Render (via render.yaml or Docker)
  • 📂 Supabase (for database backend)
